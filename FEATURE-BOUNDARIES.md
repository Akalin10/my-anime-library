# 「我的动漫库」功能边界

本文件是后续实现的功能边界清单，事实来源仅为 `REQUIREMENTS.md`。白名单是封闭清单：后续只可实现其中明确列出的功能；未列出的功能不得因“可能有用”或“页面显得更完整”而添加。若本文件与原始需求有歧义，以 `REQUIREMENTS.md` 为准。

## 功能白名单

### 产品范围与界面

- 单用户、本地优先、私人使用；打开网站后直接进入动漫库首页，不建立用户体系。（出处：REQUIREMENTS.md 第一、二章）
- 产品核心用途仅为搜索并导入动漫资料、查看个人收藏的动漫海报和资料、标记“在看”或“已看完”。（出处：REQUIREMENTS.md 开篇、第一章）
- 只提供动漫库首页、搜索并导入动漫界面、设置界面三个主要界面。（出处：REQUIREMENTS.md 第五章）
- 动漫详情只在居中模态窗口中展示，不建立独立详情页，也不改变首页布局。（出处：REQUIREMENTS.md 第五、七章）

### 动漫库首页

- 首页采用左侧导航栏、顶部搜索和操作区域、状态筛选区域、动漫海报网格组成的海报墙布局。（出处：REQUIREMENTS.md 第六章）
- 左侧导航只提供“全部”“在看”“已看完”“设置”；允许显示来自真实数据库查询的数量。（出处：REQUIREMENTS.md 第 6.1 节）
- 本地搜索只匹配已经导入动漫的中文标题、原始标题、英文标题和别名。（出处：REQUIREMENTS.md 第 6.2 节）
- “添加动漫”打开搜索并导入动漫界面。（出处：REQUIREMENTS.md 第 6.2 节）
- 状态筛选只提供“全部”“在看”“已看完”。（出处：REQUIREMENTS.md 第 6.3 节）
- 排序只提供“最近添加”“标题”“上映年份”，默认“最近添加”；标题支持中文，缺失年份排在最后。（出处：REQUIREMENTS.md 第 6.4 节）
- 海报卡片只常显海报图片、动漫标题、当前状态；点击海报主体打开详情模态窗口。（出处：REQUIREMENTS.md 第 6.5 节）

### 状态与详情

- 动漫状态只能是 `WATCHING` 和 `COMPLETED`，界面分别显示“在看”和“已看完”；每个导入条目必须选择其中之一。（出处：REQUIREMENTS.md 第三章）
- 导入默认状态允许为“在看”，但用户可在导入前切换。（出处：REQUIREMENTS.md 第三章、第 8.5 节）
- 详情模态窗口支持关闭按钮、ESC、点击遮罩关闭、内部滚动、背景滚动锁定及响应式布局。（出处：REQUIREMENTS.md 第七、二十八章）
- 详情只显示中文标题、原始标题、上映年份、动画类型、总集数、制作公司、简介、当前状态、所属系列、相关作品、更换封面、删除条目和关闭。（出处：REQUIREMENTS.md 第 7.1 节）
- 用户可在详情中切换“在看”和“已看完”，修改立即保存并同步首页卡片与筛选数量。（出处：REQUIREMENTS.md 第 7.2 节、第二十七章流程四）
- 详情可按真实关系展示正传、前传、续作、OVA/OAD、剧场版、特别篇、总集篇、外传、衍生作品及其他相关作品。（出处：REQUIREMENTS.md 第 7.3 节）
- 相关作品只显示小型封面、标题、年份、类型及“已添加/未添加”状态。（出处：REQUIREMENTS.md 第 7.3 节）
- 点击已导入的相关作品可在当前模态窗口切换详情；未导入作品可打开简单导入确认界面。（出处：REQUIREMENTS.md 第 7.3 节）
- 用户可在确认后删除本地动漫条目；删除范围仅限本地收藏数据和本地自定义封面。（出处：REQUIREMENTS.md 第 7.4 节）

### 搜索与导入

- 搜索并导入界面使用全屏模态窗口或覆盖当前页面的大型对话框。（出处：REQUIREMENTS.md 第八章）
- 外部搜索接受中文名、日文名、英文名和别名；支持点击搜索或使用 400–600ms 防抖自动搜索，并显示真实加载状态。（出处：REQUIREMENTS.md 第 8.1 节）
- 搜索结果仅来自数据源返回的真实条目和真实关联关系，尽可能覆盖季度、OVA、OAD、特别篇、总集篇、剧场版、外传和衍生动画。（出处：REQUIREMENTS.md 第 8.2 节）
- 仅在数据源关系明确时按正传、OVA/OAD、剧场版、特别篇、总集篇、外传/衍生等分组；无法确定的条目进入“其他搜索结果”。（出处：REQUIREMENTS.md 第 8.3 节）
- 搜索结果卡片显示小型海报、中文标题、原始标题、年份、类型、总集数、数据来源、导入状态和选择框。（出处：REQUIREMENTS.md 第 8.4 节）
- 用户可单选、多选、取消选择和导入选中条目；不默认全选，也不自动导入相关作品。（出处：REQUIREMENTS.md 第 8.4 节）
- 用户可为全部选中作品统一设置状态，也可逐项设置状态。（出处：REQUIREMENTS.md 第 8.5 节）
- 导入流程获取完整资料、最佳可用海报和相关关系，完成标准化、重复检查、封面本地下载、数据库保存并返回真实结果。（出处：REQUIREMENTS.md 第 8.6 节）
- 批量导入允许单项失败而不使全部失败，并逐项报告失败原因及成功/失败数量。（出处：REQUIREMENTS.md 第 8.6 节）

### 外部数据源

- 使用可扩展的数据源适配器，初始按轮次接入 Bangumi、AniList 和 TMDB；Bangumi/AniList 用于动画条目与系列关系，TMDB 用作电影、剧场版、海报或背景资料补充。（出处：REQUIREMENTS.md 第九章；具体接入顺序以 Project-prompts.md 为准）
- 数据源提供统一的搜索、详情、关系和候选海报能力，并标准化来源、外部 ID、各语言标题、别名、年份、类型、集数、制作公司、简介、海报和关系。（出处：REQUIREMENTS.md 第 9.1 节）
- 已启用数据源可并行搜索；依据标题、年份、类型、集数、外部 ID 和明确跨平台 ID 谨慎去重，并保留所有来源信息。（出处：REQUIREMENTS.md 第 9.2 节）
- 单个数据源失败时显示该来源失败、允许重试，并让其他数据源继续工作。（出处：REQUIREMENTS.md 第 9.3 节）
- 搜索和详情请求使用有合理过期时间的适度缓存，避免短时间重复请求。（出处：REQUIREMENTS.md 第 9.4 节）

### 封面

- 每部动漫支持数据源默认封面、用户上传本地封面、通过图片 URL 添加封面以及恢复数据源默认封面。（出处：REQUIREMENTS.md 第十章）
- 导入时优先获取高质量竖版海报，并尽量把远程封面下载到本地。（出处：REQUIREMENTS.md 第 10.1 节）
- 封面管理提供上传本地图片、使用图片 URL、恢复默认封面和取消；支持 JPG、JPEG、PNG、WebP，上传前预览并经确认保存。（出处：REQUIREMENTS.md 第 10.2 节）
- 图片 URL 由服务端校验 URL、响应类型、大小和安全风险，成功后下载到本地。（出处：REQUIREMENTS.md 第 10.3 节）
- 封面显示优先级固定为用户自定义本地封面、已下载默认本地封面、数据源远程封面、中性默认占位图。（出处：REQUIREMENTS.md 第 10.4 节）
- 海报以 2:3 比例、`object-fit: cover` 展示，不拉伸，失败时显示统一占位图并使用懒加载。（出处：REQUIREMENTS.md 第 10.5 节）

### 设置

- 设置仅包含启用的数据源、默认数据源优先级、数据源 API 密钥、海报本地保存目录、清理搜索缓存和查看数据库位置。（出处：REQUIREMENTS.md 第十一章）

### 真实状态、错误与基础体验

- 空库、无搜索结果、数据源请求失败和封面加载失败均显示需求指定的真实空状态、错误状态、重试入口或中性占位图。（出处：REQUIREMENTS.md 第十七章）
- 数据缺失时显示“暂无资料”，不得由 AI、常识、其他作品或不确定来源补全。（出处：REQUIREMENTS.md 第二十五章）
- 本地 SQLite 保存动漫资料，本地目录保存封面；首页只读取本地数据库，外部数据源仅在搜索、导入和刷新资料时调用。（出处：REQUIREMENTS.md 第 18.1 节、第二十二章）
- 服务端接口提供本地列表、详情、导入、状态修改、删除、封面上传、URL 封面、恢复默认封面、外部搜索和数据源状态能力，统一校验输入并返回可读错误。（出处：REQUIREMENTS.md 第二十章）
- 文件上传、远程图片、密钥、数据库查询和文本渲染必须满足需求列出的基础安全措施。（出处：REQUIREMENTS.md 第二十一章）
- 海报墙使用图片懒加载、合理尺寸、缩略图与缓存；搜索防抖和缓存；详情按需加载；数据库使用合理索引。（出处：REQUIREMENTS.md 第二十二章）
- 支持桌面、平板、手机响应式布局及适合触控的移动端交互。（出处：REQUIREMENTS.md 第十五、二十八章）
- 支持按钮标签、图片 alt、模态窗口语义与焦点管理、键盘操作、清晰焦点、足够对比度、非纯颜色状态表达和 `prefers-reduced-motion`。（出处：REQUIREMENTS.md 第二十三章）

第三十三章同时禁止：

- 保留任何没有在 `REQUIREMENTS.md` 中明确列出的功能。

- 为了填满页面而添加内容；页面允许大量留白，功能允许很少。
- 在对某项功能是否应添加存在疑问时添加该功能。
- 在数据源缺失资料时猜测或补全，而不是显示“暂无资料”。
- 在无法确认两个搜索结果是否为同一作品时强制合并。
- 在无法确认作品关系类型时强行分类，而不是放入“其他搜索结果”。
- 猜测、编造或使用预设数据制造完整感。

## 边界执行原则

- 后续每轮开始前均应对照本文件与 `REQUIREMENTS.md` 的对应章节。
- 白名单之外的功能默认禁止；新增或改变功能边界必须先修改原始需求并建立单独、明确的执行轮次。
- 第三十三章中的英文名称按原需求约束字段、按钮、页面和组件；不得以改名方式规避审计。
- 数据不确定时保留为空或显示“暂无资料”，关系不确定时不合并、不强行分类。
